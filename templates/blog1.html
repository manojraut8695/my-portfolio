Unveiling Customer Sentiments:

A Deep Dive into Amazon Alexa Reviews using Data Science

Introduction

In today's world of advanced smart technology, Amazon Alexa is a popular virtual assistant that easily fits into our daily lives. As this high-tech tool becomes more and more a part of our routines, it's important to understand how people feel about it. This article takes a close look at the field of data science and how it can be used to analyze the feelings expressed in customer reviews of Amazon Alexa. We'll use the capabilities of machine learning and deep learning to dive deep into these reviews and discover important insights.

What is Sentiment Analysis?

Imagine being able to understand not only the words in a digital document but also the feelings and attitudes they express. This is what sentiment analysis does. It's a complex process in the field of natural language processing. It uses advanced computer algorithms to figure out the emotional tone of written text, like whether it's positive, negative, or neutral. Basically, it helps us see how a lot of different people feel about something.

Preprocessing

When we start to analyze data, we need to really look closely at the information we have. Exploring the data is like having a map that helps us understand it better. We examine how the data is organized, figure out how big it is, and check if there are any strange things in it.

After exploring the data we find that there is a class imbalance in the data. 92% of reviews are positive and only 8% of reviews are negative

Dealing with imbalanced data, where one class (in this case, positive sentiment) significantly outweighs the other (negative sentiment), is a common challenge in machine learning. This would mean that the model might perform very well on predicting positive sentiment but poorly on detecting negative sentiment.

TO address this issue I implimented Resampling technique and weighted loss function.

  • Resampling comes with two steps Undersampling: Mention that we reduced the number of samples in the majority class (positive sentiment) to balance it with the minority class (negative sentiment) and Oversampling: Alternatively, we could explain that we increased the number of samples in the minority class. we may have done this by duplicating or generating synthetic samples for the negative sentiment class.
  • Weighted functions are used in the training phase to assign more importance to the minority class. In our case, we assigned a higher weight to the negative sentiment class during training. This means that errors made on the negative sentiment class will have a greater impact on the model's performance metrics than errors on the positive sentiment class.

    Data Cleaning

    Before we start analyzing the feelings in a piece of text, it's really important to make sure the text is tidy. This means getting rid of things like dots, commas, and strange symbols, as well as stopwords and taking out words that don't add much meaning to the analysis. Cleaning up the text makes sure that the analysis we do afterwards is done on clear and meaningful words. This is the most important step for any texual analysis project.

    Implementation of Classification Machine Learning Models

    Befome Implementating model we split the data into training and testing sets that is 80% of data for training and 20% data for testing.

    We used different types of models to classify things in our project. Each model has its own strengths and unique features. We tried models like Logistic Regression, Naive Bayes (including BernoulliNB, GaussianNB, and MultinomialNB), Support Vector Machine (SVM), Decision Tree, and Random Forest. We trained these models carefully and checked how well they worked.

    We tested all of these models and got different accuracy for different models mentioned below:

    Logistic Regression: Accuracy - 0.917
    Naive Bayes (BernoulliNB): Accuracy - 0.905
    Naive Bayes (GaussianNB): Accuracy - 0.552
    Naive Bayes (MultinomialNB): Accuracy - 0.916
    SVM: Accuracy - 0.927
    Decision Tree: Accuracy - 0.932
    Random Forest: Accuracy - 0.938

    The Random Forest model performed the best, with an accuracy of 0.938. After carefully trying out different options, we found that the best settings for the model were {'max_depth': None, 'min_samples_split': 2, 'n_estimators': 100}. This led to a very good accuracy of 0.937.

    Deep Learning Techniques:

    We started exploring deep learning, which involves using Artificial Neural Networks (ANN) and Recurrent Neural Networks (RNN). These methods help us find complex patterns in data and offer a special way to understand it.

    ANN: Accuracy - 0.932, Loss - 0.499
    RNN: Accuracy - 0.932, Loss - 0.315

    After evaluating various machine learning (ML) and deep learning (DL) models, we found that the Random Forest model outperformed all others with an accuracy of 93.8%. The RNN (Recurrent Neural Network) model also performed well, albeit slightly less accurately at 93.7%. Despite Random Forest's superior accuracy, we decided to proceed with the RNN model for further analysis. The rationale behind this decision is that ML models sometimes struggle when stop words are removed from text data. For instance, consider two reviews:

    1. "The product is good"
    2. "The product is not good."

    If we remove stop words, both sentences become "product good," and the ML model might mistakenly classify both as positive sentiments. However, the RNN model operates differently from ML models. It takes the context and phrases within reviews into account, allowing it to make more nuanced predictions about positive and negative sentiments.

    Deployment Using Flask Framework

    We used a tool called Flask to make our best model, the RNN (Recurrent Neural Network), work in the real world. This means we can now instantly figure out how people feel about Amazon Alexa by looking at new reviews. This helps us turn our data analysis into useful information that we can act on.
